数当てゲーム

このゲームは、1から10までのランダムな整数をプレイヤーが予想して当てる数当てゲーム。 プレイヤーは入力ボックスに自分の予想した数を入力し、「回答」ボタンをクリックすることで予想を送信する。 予想のたびに回数がカウントされ、最大3回まで挑戦できる。 予想が当たれば「正解です、おめでとう!」と表示されゲームは即終了する。 また、外れた場合は「もっと大きい」または「もっと小さい」とヒントが表示され、プレイヤーが次の予想をしやすいように工夫されている。 3回目の予想が外れた時点でゲーム終了となり、正解の数が表示される。 inputやbutton要素を正しく操作して、回数や結果を表示する処理に苦労した。 回数や結果の表示をリアルタイムで更新するために用いた DOM の操作にも苦戦した。

数当てゲームのページ: kazuate.html

各授業回の学習事項

第1回 WWW と HTML

webページを構成するHTML文書の使い方を学んだ。 また、pやstorongのブロック要素とインライン要素について学んだ。 HTML全体の構造を組み立てるうえで、これらの要素がどのように使われるか、それぞれが持つ役割についても学ぶことができた。

演習の成果: elements.html

第2回 CSS による Web ページの装飾

CSSとHTMLを組み合わせてWebページのデザインを整える方法について、3通りの記述方法を学んだ。 まず1つは外部スタイルシートで、CSSを別ファイルで用意し、HTMLファイルの head 内で link タグを使って読み込む方法。 2つ目は内部スタイルシートで、HTMLファイルの head 要素内に style タグを作成し、その中に直接CSSを記述する方法。 3つ目はインラインスタイルで、HTMLタグのstyle属性に直接CSSを書く方法。

演習の成果: browser-intro.html

第3回 JavaScript の基本

Webページに動きのある機能を実現するために、JavaScriptを使用する方法を学んだ。 JavaScriptは、HTMLファイルとは別にjsファイルを作成し、それをHTMLの head 要素や body 要素の末尾に script タグを使って読み込む。 こうすることで、HTMLと動作JavaScriptを分離し、Webページを作成できる。 。

演習の成果obj-practice.html

第4回 DOM と JavaScript

ボタンを押すことでページの内容が変化する処理をJavaScriptで学んだ。 JavaScriptとDOMを組み合わせて使うことが重要である。 DOMは、HTML文書内の各要素をJavaScriptのオブジェクトとして対応づける仕組みであり、これを利用することでページの要素を直接操作できる。

演習の成果: browser-intro.html

第5回 イベント処理

データをコンソールに出力するのではなく、ページ上に表示できるようにした。 これにより、ユーザーが画面上で検索結果などの情報を確認できるわかりやすいwebページが完成した。。 ページ上に要素を表示するために、要素名を使って新しい要素を作成する。 そして、要素を用いて必要な属性を設定し、内容を追加する。 また、不要になった要素は 要素.remove() で削除することができる。

演習の成果: elements.html

授業を通して、最初は見慣れない構文や文法に戸惑いながらも、JavaScriptやHTML、DOMの概念が結びついていく過程には発見と学びがあり、とても面白く感じました。 初めて記述した addEventListener が正しく動作したときの感動や、DOMの構造を理解し始めたことで、HTMLの要素を自在に操作できるようになっていったことは、特に印象的でした。 最終課題では、これまでに学習した内容を活用し、天気情報APIからデータを取得してWebページ上に表示するアプリケーションの制作に取り組みました。 この課題を通して、Webページがどのように動的に情報を取得・表示しているのかを、実際に手を動かしながら具体的に学ぶことができました。 苦労した点は、APIから取得するJSONデータの扱い方でした。 データ構造の中にはオブジェクトや配列が混在しており、たとえば data.weather[0].description のように、少し複雑な書き方が必要でした。 最初は正しい参照方法が分からず、多くのエラーに悩まされましたが、console.log() を使ってデータ構造を可視化しながら、一つずつ確認して正しい記述方法を習得していきました。 また、JavaScriptの非同期通信の仕組みも、初めは理解が難しかったです。 データを取得する前に画面描画が始まってしまい、結果として何も表示されないという問題が何度も発生しました。 このような状況に対しては、then() や catch() を使ったPromiseの記述方法を理解し、非同期処理の流れを正しく制御できるよう工夫しました。 特に印象に残ったのは、HTMLとJavaScriptを連携させることで、静的なWebページがユーザーと対話できる動くアプリケーションへと変わる点です。 入力フォームから値を取得し、サーバーと通信を行い、受け取ったデータを画面に表示する一連の流れを自分の力で構築できたことで、Webアプリケーションの仕組みをより深く理解することができました。 特に、ページのリロードなしに内容が書き換わるという体験は新鮮で、動的なWebの可能性を実感するきっかけとなりました。 また、実装の過程ではトラブルやエラーも多く発生しましたが、ブラウザのデベロッパーツールや console.log() を用いて原因を一つずつ調べ、修正するという工程を繰り返しました。 今後は、今回学んだ基礎を活かして、より複雑で難しい処理にもチャレンジしていきたいと考えています。 今回の授業で得た知識や経験は、今後の学習や制作活動の中でも確実に役立つと感じました。 単なるプログラミングの演習にとどまらず、Web開発のプロセスを体験できたことで、大きな自信につながりました。
